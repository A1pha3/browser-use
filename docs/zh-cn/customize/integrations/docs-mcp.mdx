## 概述

browser-use 文档 MCP 服务器为 Claude Code 和其他 MCP 兼容客户端提供对 browser-use 文档的只读访问。这使 AI 助手在帮助你编写代码时能够深入了解 browser-use 库。

<Note>
想要给助手添加 browser-use 能力？请查看我们的<a href="/customize/integrations/mcp-server">浏览器自动化 MCP</a>。
</Note>

## 快速开始

将文档服务器添加到你的编码助手：

<Tabs>
<Tab title="Claude Code">
```bash
claude mcp add --transport http browser-use https://docs.browser-use.com/mcp
```
</Tab>

<Tab title="Cursor">
添加到 `~/.cursor/mcp.json`：

```json
{
  "mcpServers": {
    "browser-use-docs": {
      "url": "https://docs.browser-use.com/mcp"
    }
  }
}
```
</Tab>

<Tab title="Codex">
添加到 `~/.codex/config.toml`：

```toml
[mcp_servers.browser-use-docs]
url = "https://docs.browser-use.com/mcp"
```
</Tab>

<Tab title="Windsurf">
添加到 `~/.codeium/windsurf/mcp_config.json`：

```json
{
  "mcpServers": {
    "browser-use-docs": {
      "serverUrl": "https://docs.browser-use.com/mcp"
    }
  }
}
```
</Tab>
</Tabs>

这使你的 AI 编码助手能够在回答问题或帮助实现时访问 browser-use 文档。

## 提供的内容

文档 MCP 服务器为 AI 助手提供以下访问：

- API 参考和使用模式
- 配置选项和参数
- 最佳实践和示例
- 故障排除指南
- 架构说明

**示例交互：**
```
"如何在 browser-use 中配置自定义工具？"

"有哪些可用的代理参数？"

"展示如何使用云端浏览器。"
```

Claude Code 现在可以使用最新的文档上下文回答这些问题。

## 工作原理

MCP 服务器提供只读文档接口：
- 通过 HTTP 提供 browser-use 文档
- 无浏览器自动化功能（请参阅 [MCP 服务器](/customize/integrations/mcp-server)）
- 轻量级且始终可用
- 无需 API Key 或配置
