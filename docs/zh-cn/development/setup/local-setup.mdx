## 欢迎来到 Browser Use 开发！

```bash
git clone https://github.com/browser-use/browser-use
cd browser-use
uv sync --all-extras --dev
# 或使用 pip：pip install -U git+https://github.com/browser-use/browser-use.git@main
```

## 环境配置

设置你的环境变量：

```bash
# 复制示例环境文件
cp .env.example .env

# 设置日志级别
# BROWSER_USE_LOGGING_LEVEL=debug
```

## 辅助脚本

常用开发任务脚本：

```bash
# 完整设置脚本 - 安装 uv，创建虚拟环境，安装依赖
./bin/setup.sh

# 运行所有预提交钩子（格式化、 linting、类型检查）
./bin/lint.sh

# 运行 CI 中执行的核心测试套件
./bin/test.sh
```

## 运行示例

```bash
uv run examples/simple.py
```

## 开发工作流

### 1. 创建功能分支

```bash
git checkout -b feature/your-feature-name
```

### 2. 编写代码

按照项目编码规范编写代码。

### 3. 运行测试

```bash
./bin/test.sh
```

### 4. 提交更改

```bash
git add .
git commit -m "feat: 添加新功能"
git push origin feature/your-feature-name
```

## 代码规范

### 格式化

项目使用以下工具保持代码质量：

- **Black** - 代码格式化
- **Ruff** - 代码 linting
- **mypy** - 类型检查

运行格式化：

```bash
./bin/lint.sh
```

### 类型注解

所有公开 API 必须包含类型注解：

```python
from typing import List, Dict

def process_items(items: List[Dict[str, str]]) -> List[str]:
    """处理项目列表"""
    return [item["name"] for item in items]
```

### 文档字符串

使用 Google 风格的文档字符串：

```python
def function_name(param1: str, param2: int = 10) -> bool:
    """简要描述函数功能。

    Args:
        param1: 参数说明
        param2: 参数说明，默认值为 10

    Returns:
        返回值说明

    Raises:
        ValueError: 异常条件说明
    """
    pass
```

## 测试

### 编写测试

测试文件应位于 `tests/` 目录下：

```python
def test_example():
    """测试示例"""
    assert True
```

### 运行特定测试

```bash
uv pytest tests/test_example.py
```
